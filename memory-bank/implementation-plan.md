# 电路信息管理系统 - 实施计划

本计划为 AI 开发者提供分步实施指令，严格遵守 `tech-stack.md` 中的规范。

## 阶段 1：后端基础架构搭建

### 步骤 1.1：初始化后端项目环境
- **指令**：在 `backend/` 目录下初始化 Node.js 项目。安装 `express`, `sequelize`, `sqlite3`, `dotenv`, `cors`, `winston`, `nodemon` 等核心依赖。`mysql2` 仅作为一次性迁移脚本依赖保留。
- **验证**：运行项目并确认 `package.json` 包含以上依赖，且入口文件 `app.js` 能在 6100 端口正常启动。

### 步骤 1.2：配置环境变量与数据库连接
- **指令**：创建 `.env` 文件，配置开发环境的数据库连接参数。在 `src/config/database.js` 中使用 Sequelize 初始化数据库连接，并导出实例。
- **验证**：通过日志输出确认 "Database connection has been established successfully"。

### 步骤 1.3：实现统一响应与错误处理
- **指令**：在 `src/utils/response.js` 中封装统一的成功响应格式。在 `src/middlewares/error-handler.js` 中实现全局错误处理中间件，捕获并返回符合规范的错误 JSON。
- **验证**：访问测试路由，确认接口返回的 JSON 结构符合 `tech-stack.md` 中的规范。

---

## 阶段 2：数据库模型与关联定义

### 步骤 2.1：定义核心实体模型（第一部分）
- **指令**：在 `src/models/` 下创建 `customer.js` 和 `topology.js`。根据设计文档定义字段（名称、联系方式等），确保文件名使用 kebab-case。
- **验证**：通过 Sequelize 的 `sync` 功能在 SQLite 中成功创建 `customers` 和 `network_topologies` 表。

### 步骤 2.2：定义设备与连接模型（第二部分）
- **指令**：在 `src/models/` 下创建 `device.js`, `port.js`, `connection.js`。定义设备类型枚举、坐标字段、端口速率等。
- **验证**：确认数据库中已生成对应的表，且外键约束符合 `design-document.md` 中的逻辑。

### 步骤 2.3：定义电路模型与模型关联汇总
- **指令**：创建 `circuit.js` 模型。在 `src/models/index.js` 中配置所有模型之间的 `hasMany` 和 `belongsTo` 关联关系，并导出。
- **验证**：通过 API 接口查询一个客户，确认能够级联查出其下的拓扑和电路数据。

---

## 阶段 3：后端 API 开发（按功能模块）

### 步骤 3.1：实现客户管理 API
- **指令**：创建 `customer-service.js`, `customer-controller.js` 和 `customer-routes.js`。实现客户的增删改查。
- **验证**：使用 Postman 发送 POST 请求创建客户，发送 GET 请求获取客户列表，确认返回结果正确。

### 步骤 3.2：实现拓扑与设备管理 API
- **指令**：实现拓扑的 CRUD 接口。实现设备的添加、删除以及位置（坐标）更新接口。
- **验证**：为一个拓扑添加多台设备，并调用更新位置接口，确认数据库中的 X/Y 坐标已改变。

### 3.3：实现端口与连接管理 API
- **指令**：实现设备端口管理接口。实现连接（Connection）接口，用于关联两个特定的端口，并存储光纤信息。
- **验证**：创建一个连接，指定源端口 ID 和目标端口 ID，确认连接记录成功指向两个端口。

### 步骤 3.4：实现电路管理与搜索 API
- **指令**：实现电路的录入和更新。实现基于电路编号和 IP 地址的搜索接口。
- **验证**：输入部分电路编号进行模糊搜索，确认接口返回匹配的电路数组。

---

## 阶段 4：小程序基础开发

### 步骤 4.1：初始化小程序项目
- **指令**：创建微信原生小程序项目，配置 `app.json` 基础页面路径。通过 npm 安装并引入 `Vant Weapp` 组件库。
- **验证**：在首页放置一个 Vant 的 Button 组件，确认能正常显示样式。

### 步骤 4.2：封装网络请求工具
- **指令**：在 `utils/request.js` 中基于 `wx.request` 封装 Promise 风格的 HTTP 客户端。处理统一的 BaseURL 和错误提示。
- **验证**：调用封装好的请求工具访问后端健康检查接口，确认能收到成功回调。

### 步骤 4.3：定义 API 请求模块
- **指令**：在 `api/` 目录下按模块（customer, topology, circuit）拆分 API 定义文件，导出各接口的调用函数。
- **验证**：确认 `customer-api.js` 中包含了获取列表和详情的所有函数定义。

---

## 阶段 5：小程序功能页面实现

### 步骤 5.1：实现客户与电路列表页
- **指令**：开发客户列表页和电路列表页，使用 Vant 的 `Cell` 和 `Search` 组件。实现下拉刷新和上拉加载。
- **验证**：打开页面时能看到从后端加载的测试数据，且搜索框输入时能实时过滤（或触发接口搜索）。

### 步骤 5.2：实现数据录入表单（客户/电路）
- **指令**：创建 `customer/form` 和 `circuit/form` 页面。使用 Vant 的 `Field`, `Picker`, `DatetimePicker` 等组件收集信息。
- **验证**：填写表单并提交，确认后端数据库中生成了对应的记录。

---

## 6：拓扑可视化核心开发

### 步骤 6.1：实现 Canvas 拓扑图基础绘制
- **指令**：在 `components/topology-canvas/` 中使用 Canvas 2D API 绘制设备（图标+文字）和连接线。
- **验证**：传入模拟的设备坐标数据，确认组件能在 Canvas 上正确渲染出设备图标和连线。

### 步骤 6.2：实现设备拖动与布局保存
- **指令**：在 Canvas 组件中监听 `touchstart`, `touchmove`, `touchend` 事件。实现拖拽设备更新 X/Y 坐标，并在拖拽结束时调用后端接口保存。
- **验证**：真机预览时拖动设备，刷新页面后设备位置保持在拖动后的地方。

### 步骤 6.3：实现自动布局算法（后端与前端配合）
- **指令**：在后端 `services/layout-service.js` 实现简单的层次布局算法。前端调用该算法获取初始位置。
- **验证**：对于新创建的拓扑，设备能按层级整齐排列，而不是重叠在 (0,0) 位置。

---

## 阶段 7：电路链路绑定与高亮

### 步骤 7.1：实现电路详情中的拓扑展示
- **指令**：在电路详情页引入拓扑 Canvas 组件。页面加载时获取该电路所属客户的拓扑数据。
- **验证**：进入电路详情，能看到对应的组网拓扑图。

### 步骤 7.2：实现链路高亮逻辑
- **指令**：为 Canvas 组件添加 `highlightConnectionId` 属性。在绘制连线时，如果连接 ID 匹配，则应用加粗和特定的高亮颜色。
- **验证**：打开特定电路详情，该电路绑定的那条物理链路在图中明显区别于其他连线。

### 步骤 7.3：实现点击交互与详情弹窗
- **指令**：在 Canvas 中实现点击检测。点击设备或连线时，弹出 Vant 的 `Popup` 显示详细信息（如光纤芯数、设备型号）。
- **验证**：点击图中连线，确认弹窗显示了正确的光纤信息。

---

## 阶段 8：系统优化与部署

### 步骤 8.1：数据缓存与性能优化
- **指令**：在小程序端使用 `wx.setStorage` 缓存客户列表等不常变动的数据。在 Canvas 绘制中使用防抖（debounce）。
- **验证**：快速切换页面时，列表能瞬间显示（来自缓存），Canvas 拖动时无明显掉帧。

### 步骤 8.2：后端部署与 Nginx 配置
- **指令**：在 Linux 服务器上安装 Node.js。使用 `npm run init-db` 初始化 SQLite 数据库文件，使用 PM2 启动后端服务。配置 Nginx 转发 API 请求并启用 HTTPS。
- **验证**：通过生产环境域名访问 API，确认返回结果正确。

### 步骤 8.3：小程序发布准备
- **指令**：在微信开发者工具中进行“上传”，填写版本号。在管理后台配置服务器域名。
- **验证**：在小程序管理后台提交审核，确认所有页面在真机预览模式下功能完整。
